ВАЖНО: ПЕРЕД ЗАПУСКОМ НАДО ЗАПУСТИТЬ JSON-SERVER --WATCH DB.JSON и потом уже программу.

Программа в ветке master. На первой странице выбираем язык программы и переходим на страницу авторизации. 
Вводим в форму авторизации данные и отравляем при помощи fetch (адрес и т.д. еще не заполнено). Если данные почты и пароля уже существуют и совпадают с базой, входим в аккаунт, если нет то повторяем заново. (Пока логика не дописана т.к. пока не знаю какую инфу ты мне вернешь с сервера).
При успешном входе в аккаунт перемещаемся на страницу ввода упражнений. Логика здесь такова: после заполнения поля "упражнение" и снятии фокуса идет запрос на сервер с данным названием упражнения и сервер возвращает массив с данными по форме как в файле db.json. Сейчас логика получения оформлена через RTK Query в автоматическом режиме, но когда ты сделаешь базу данных хочу переписать логику получения данных по упражнениям также через RTK Query, но уже как пангинация данных (чтобы не грузить лишней информации с сервера). Да еще думаю надо сделать загрузку массива например длиной не более 10 элементов, а если их больше, то я предусмотрю дозагрузку). Далее в программе полученные данные обрабатываются и выводятся определнным образом.
После заполнения полей и при нажатии на кнопку "Сохранить" собранные данные отправляются на сервер для записи. Пока запись идет в файл db.json
